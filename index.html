Session & Memory

/init
Initialize project with CLAUDE.md. Creates your project's persistent instruction file read at every session start.
▸ example
/init
◈ output
✓ CLAUDE.md created (18 lines) Detected: Node.js · TypeScript · Jest Added: project structure, test cmds, style notes Claude will read this at every session start.
/memory
View and manage auto-memory files. Use # prefix in the REPL to add things to memory without editing files.
▸ example
/memory
◈ output
Memory: ~/.claude/projects/.../memory/ user_role.md senior fullstack dev feedback_testing.md run tests before commit project_context.md auth refactor (compliance) [E]dit [D]elete [N]ew
/resume
Resume a previous session. /resume 1 for most recent, /resume name by name. Sessions are resumable up to 7 days.
▸ example
/resume auth-refactor
◈ output
Resuming: "auth-refactor" (2h ago · 34 msgs) Last action: edited src/auth/middleware.ts Context restored. Continue where you left off.
/rename
Name the current session for easy resuming later.
▸ example
/rename auth-refactor
◈ output
Session renamed → "auth-refactor" Resume later with: /resume auth-refactor
/clear
Clear conversation history. Keeps files and memory intact — only the chat context is wiped.
▸ example
/clear
◈ output
Conversation cleared (47 messages removed). Files, memory, and tool state are intact.

Condense conversation history to free up context. Optionally focus: /compact focus on tests

Model & Effort

/model
Switch model mid-session. Options: sonnet (default), opus, haiku, fable. Also Alt+P while typing.
▸ example
/model opus
◈ output
Model → claude-opus-4-8 Effort: medium Best for: complex reasoning, agentic tasks
/effort
Set reasoning effort: low, medium, high, xhigh. Use xhigh with Opus for hardest agentic work.
▸ example
/effort xhigh
◈ output
Effort → xhigh Claude will reason more deeply before responding. Recommended pairing: /model opus
/fast
Toggle fast output mode — uses Opus with optimized output speed.
▸ example
/fast
◈ output
Fast mode ON Using Opus 4.8 with optimized output speed. Toggle off: /fast

Autonomous Work

/loop agentic
Run a prompt or command on a repeating interval. /loop 3h check PRs, /loop 5m /my-skill, or bare /loop for self-paced maintenance. Stop with /loop stop.
▸ example
/loop 5m check for new failing tests
◈ output
Loop started ↺ Interval: every 5 minutes Task: check for new failing tests Wakeup #1: 14:47 Stop with: /loop stop or Ctrl+C
/goal v2.1.139+
Set a completion condition; Claude keeps going autonomously until the condition is met. Uses a fast model to evaluate after each turn.
▸ example
/goal all tests pass
◈ output
Goal set: "all tests pass" Running autonomously... ✗ Turn 1: 3 tests failing in auth.test.ts → Fixing middleware null check... ✓ Turn 2: All 47 tests passing Goal reached in 2 turns. Stopping.
/plan
Enter plan mode — Claude outlines the approach before touching any files. Best before large refactors.
▸ example
/plan refactor the auth module
◈ output
Plan: Refactor auth module Phase 1 — Audit (read-only) • Read src/auth/*.ts (6 files) • Identify session token issues Phase 2 — Restructure • Split middleware.ts → validator + session • Update imports in 4 consumers Phase 3 — Test: jest --testPathPattern=auth Approve to begin → [Y/n]
/batch
Run batch operations across multiple files or targets in parallel.
▸ example
/batch add JSDoc to all files in src/utils/
◈ output
Batch: 8 files in src/utils/ ✓ formatDate.ts — 3 functions documented ✓ validators.ts — 7 functions documented ✓ http.ts — 4 functions documented ... (5 more) Completed in 12s · 0 errors
/tasks
List all background agents currently running in this session.
▸ example
/tasks
◈ output
Background agents (2 running) [1] ● pr-reviewer Running 1m 34s [2] ● test-watcher Running 4m 02s /tasks stop 1 to stop agent #1
/bashes
List background bash tasks spawned by Claude.
▸ example
/bashes
◈ output
Background bash tasks (1 running) [1] ● npm run build Running 0m 48s PID 83421 Output: /tasks output 1 Stop: /tasks stop 1
Tip: /loop vs /goal — use /loop when you want Claude to wake up on a schedule. Use /goal when you want Claude to keep working until a condition is true.

Custom Commands

Create a .md file in .claude/commands/ (project) or ~/.claude/commands/ (personal) and invoke it with /filename. Use $ARGUMENTS for arguments.
--- # .claude/commands/pr-review.md description: Review open PRs and post GitHub comments allowed-tools: Bash(gh pr list *), Bash(gh pr diff *), Bash(gh pr review *) model: opus --- 1. Run `gh pr list --state open --json number,title` 2. For each PR, run `gh pr diff <number>` 3. Review for: breaking API changes, missing error handling, security issues 4. Post: `gh pr review <number> --comment -b "<feedback>"`
Usage: /pr-review — or schedule it: /loop 3h /pr-review

Permission Modes

manual (default)
Prompts on first use of each tool
Normal development
acceptEdits
Auto-approves file edits, prompts for bash
Trusted projects
auto
Safety classifier reviews each action automatically
Autonomous with guardrails
plan
Read-only — no edits or execution
Analysis only
bypassPermissions
Skips all prompts entirely
CI/CD automation
Cycle modes with Shift+Tab. Set at startup: claude --permission-mode auto
